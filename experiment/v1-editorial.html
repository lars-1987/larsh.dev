SP — 01 / WEB

Shipped · Web app

SanctumPDF

Privacy-first PDF & financial-file conversion — the heavy lifting happens in the browser.

  • Client-heavy, server-light. Wasm does the PDF work in-browser; files live in memory and vanish on tab close.
  • Avoided the AGPL trap. pdf-lib + QPDF Wasm cut the payload ~7× and removed a licensing landmine.
  • 14-locale i18n grew the indexed surface from ~36 to 335+ pages.

Next.js · Tailwind · Supabase · Stripe · Vercel · Wasm

RN — 02 / iOS

Shipped · Mobile app

Renew

Skincare tracking with real analytics and an OCR → AI label pipeline.

  • OCR → Gemini extraction. Point the camera at a product; the label becomes structured, searchable INCI data — Korean and Japanese included.
  • Offline-first by architecture. A local WatermelonDB makes logging instant with no signal, then syncs to Supabase.
  • Analytics as the differentiator — skin-score trends and time-lagged ingredient correlation.

React Native · Expo · WatermelonDB · Supabase · OCR + Gemini

MS — 03 / WEB · OSS

Shipped · Open source

MetaStrip

Browser-based metadata removal — fully client-side, verifiable in the Network tab.

  • Equal-size metadata replacement. Hand-written MP4/ID3/Vorbis/RIFF parsers swap metadata for identical-length padding, so playback never breaks.
  • Structural privacy, not a promise. Zero upload, zero server, zero cache — the guarantee is the architecture.

Next.js · 100% client-side · MIT · custom binary parsers

BC — 04 / iOS 26

In development · Mobile app

Bedcheck

An iOS accountability app for your bedtime, not your wake time.

  • Built against AlarmKit before the docs existed. Read the .swiftinterface directly; surfaced silent Xcode bugs through device dogfooding.
  • Three-rail reliability. Local notifications + AlarmKit + a Live Activity — each can fail without breaking the safety net.
  • A ~7,000-line spec before code, with every accepted failure mode documented.

Swift 6 · SwiftUI · SwiftData · AlarmKit · ActivityKit · HealthKit
